# Аудит сайта Сервис.com — заметки разработчика

Полный проход по коду в текущем состоянии. Критических уязвимостей и явных
функциональных багов не найдено. Ниже — что проверено, что исправлено и
рекомендации.

## ✅ Исправлено в этом аудите
1. **Тест рассинхронился с показом остатков.** `tests/api.test.js` проверял, что
   публичный API скрывает остаток, — но мы намеренно начали его показывать.
   Тест приведён в соответствие (теперь ожидает остаток в публичном API).
2. **Бэкап БД при WAL.** `scripts/backup.js` копировал `data.sqlite` без сброса WAL —
   свежие записи из `-wal` могли не попасть в копию. Добавлен `wal_checkpoint`
   перед копированием → бэкап консистентен.

## Безопасность — сильная
- Пароль админа: scrypt-хэш + сравнение в постоянное время (timingSafeEqual).
- JWT для админ-сессии; в проде сервер отказывается стартовать с дефолтными секретами.
- Rate-limit: вход 10/15мин, импорт 120/15мин, общий API 300/мин, заявки 20/мин.
- Helmet + CSP: `img-src https:` (фото Al-Style грузятся), скрипты — self + Метрика,
  `objectSrc 'none'`, `baseUri/frameAncestors 'self'`, upgrade-insecure-requests.
- Загрузка фото в админке: проверка magic-bytes + расширения + размера, санитизация имени.
- `/api/order` не доверяет ценам/названиям клиента — берёт товар из БД по sku/id.
- Глобальный обработчик ошибок не светит стек (только generic-сообщение).
- Эндпоинты импорта/категорий защищены IMPORT_TOKEN, админские — JWT.

Замечания (не критично):
- CSP использует `'unsafe-inline'` для скриптов — нужно из-за inline-обработчиков
  (onclick). XSS в остальном закрыт экранированием. Приемлемый компромисс.
- CORS по умолчанию открыт (если не задан `CORS_ORIGINS`). Записи защищены токеном,
  так что это не дыра, но в проде стоит указать `CORS_ORIGINS` со своим доменом.
- Остаток теперь виден в публичном API — это сделано намеренно (по запросу владельца).

## Данные и надёжность — ок
- `categories-sync` бережный: ручные настройки видимости/порядка не стираются импортом.
- При импорте вернувшийся товар снова делается видимым (`visible=1`).
- Несколько фото (галерея) — корректно (берутся из поля images Al-Style).
- БД: WAL, индексы на месте, мягкие миграции колонок.

Замечания (не критично):
- `scripts/seed.js` в INSERT не указывает новую колонку `images` — безвредно
  (seed запускается только на пустой базе, колонка необязательна).

## Производительность — ок для текущего масштаба
- `/api/products` отдаёт все видимые товары (с кэшем). При ~2300 товарах это норма.
  Если каталог вырастет до 10k+, стоит сделать пагинацию/серверную фильтрацию.
- Дашборд один раз грузит все товары и заявки в память — норма на этом масштабе.

## SEO — сильно
- Отдельные URL `/product/<sku>`, canonical, хлебные крошки.
- JSON-LD Product: sku, mpn, цена, состояние, наличие, срок цены.
- Open Graph с ценой; динамический `sitemap.xml` с фото товаров; пинг IndexNow.

## UX / консистентность — ок
- Каталог категорий слева; фильтры в кнопке у сортировки; Тип/Разрешение
  показываются только там, где есть такие товары (камеры).
- Остатки выводятся на карточках и на странице товара.
- Страницы категорий приведены в единый вид; стили в общем style.css.
- Адаптив под мобильные присутствует (медиазапросы для каталога/галереи).
