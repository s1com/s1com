# Подключение API Al-Style → каталог «Сервис.com»
### Инструкция и маппинг полей для разработчика

Цель: автоматически заливать товары из Al-Style в наш сайт (эндпоинт `/api/import`)
с актуальными ценами, остатками и фото. Запускать по расписанию (раз в день/час).

────────────────────────────────────────────────────────────────────
1. ДОСТУП К API AL-STYLE
────────────────────────────────────────────────────────────────────
- Кабинет: https://b2bportal.al-style.kz  → раздел «Интеграции (API, Виджеты)»
  ( https://b2bportal.al-style.kz/get-api/ ).
- Там получить: API-ключ (токен), URL методов, и ТОЧНЫЙ формат ответа.
- ВАЖНО: первым делом сделать один тестовый запрос и посмотреть реальный JSON —
  названия полей ниже даны по данным их витрины и требуют подтверждения по факту.

────────────────────────────────────────────────────────────────────
2. ПОЛЯ AL-STYLE → ПОЛЯ НАШЕГО КАТАЛОГА
────────────────────────────────────────────────────────────────────
Наш приёмник принимает массив товаров такого вида (POST /api/import):
  { sku, brand, model, group, cat, desc, res, price, stock, img }

Соответствие (левое — у Al-Style, по данным витрины; ПОДТВЕРДИТЬ по реальному ответу API):

  Al-Style                         →  Наш каталог     Примечание
  ───────────────────────────────────────────────────────────────────────────
  код (напр. 89408)                →  sku             УНИКАЛЬНЫЙ ключ. Стабильный
                                                       внутренний код Al-Style.
                                                       Используем его как артикул.
  Артикул-PartNumber               →  model           Заводской партномер/название
  name (полное название)           →  model или desc  Можно положить в model, а из
                                                       name выделить бренд.
  brand (бренд)                    →  brand           IronLogic, Dahua, Hikvision…
  category / категория             →  group           ЧЕРЕЗ КАРТУ КАТЕГОРИЙ (п.4)
  price (дилерская цена, тг)       →  price            ПРИМЕНИТЬ НАЦЕНКУ (п.3)!
  count / quantity / остаток       →  stock           Число на складе
  характеристики / specs           →  desc, res       Описание и разрешение/хар-ки
  код → URL картинки               →  img             https://img.al-style.kz/{код}_01.jpg

Картинки: у Al-Style фото лежат по схеме  https://img.al-style.kz/{код}_01.jpg
(например, код 89408 → https://img.al-style.kz/89408_01.jpg). Наш сайт принимает
полный URL в поле img и показывает его как есть. Если фото нет — оставить img пустым.

────────────────────────────────────────────────────────────────────
3. ⚠ ЦЕНА — ГЛАВНОЕ. НАЦЕНКА И ЗАЩИТА МАРЖИ
────────────────────────────────────────────────────────────────────
Al-Style отдаёт ДИЛЕРСКУЮ (закупочную) цену. На сайте должна стоять РОЗНИЧНАЯ.
Формула:   розница = округлить( дилерская × (1 + наценка%) )

- Наценку задаёт владелец (в скрипте параметр MARKUP, можно разную по категориям).
- ПРИНЦИП (из бизнес-разбора): не опускать розницу ниже закупа + минимальная маржа.
  Это та же логика, что вы внедрили в 1С (запрет продаж ниже себестоимости) —
  здесь она защищает маржу на витрине. Никогда не публиковать цену ниже дилерской.
- Округление: до 10–100 тг вверх (чтобы цены выглядели аккуратно).

Пример: дилерская 20 045 тг, наценка 25% → 25 056 → округлить до 25 100 тг.

────────────────────────────────────────────────────────────────────
4. КАРТА КАТЕГОРИЙ (Al-Style → наши категории)
────────────────────────────────────────────────────────────────────
Категории на нашем сайте управляются в админке. Маппинг (стартовый, дополняйте):

  Al-Style категория                          →  Наша категория (group)
  ─────────────────────────────────────────────────────────────────────
  Системы видеонаблюдения / IP видеонабл. /
    Аналоговое и HD видеонаблюдение / Сетевые камеры → Видеонаблюдение
  Системы контроля доступа (СКУД) /
    Контроллеры / Считыватели / домофоны        →  СКУД и домофония
  Охранные и пожарные системы / Аналоговая ОПС  →  Пожарная безопасность
  Активное сетевое / Коммутаторы / Wi-Fi /
    Маршрутизаторы / Трансиверы                 →  Сетевое оборудование
  Кабельные системы / витая пара / коннекторы   →  Кабельные системы
  Источники бесперебойного питания (ИБП)        →  Источники бесперебойного питания (ИБП)
  Системы хранения данных / HDD и SSD           →  Серверное оборудование и СХД
  …                                             →  (добавить по мере наполнения)

Имена «наших» категорий должны ТОЧНО совпадать с названиями в админке (раздел
«🗂 Категории»), иначе товар попадёт в категорию, которой нет. Если категории
ещё нет — сначала создать её в админке (или скрипт зальёт товар, а категорию
добавите вручную и включите галочку «Показывать»).

────────────────────────────────────────────────────────────────────
5. КАК ОТПРАВЛЯТЬ В НАШ САЙТ
────────────────────────────────────────────────────────────────────
POST  https://<наш-домен>/api/import
Заголовок:  Authorization: Bearer <IMPORT_TOKEN>   (взять из .env сайта)
Тело (JSON):
  {
    "source": "al-style",
    "products": [ { ...товар... }, ... ],   // до 5000 за раз
    "fullSync": false                        // true = деактивировать отсутствующие
  }

Поведение приёмника:
- upsert по sku (есть → обновить цену/остаток/фото, нет → создать);
- fullSync:true — товары, которых нет в выгрузке, помечаются неактивными
  (удобно для полной синхронизации раз в сутки);
- при добавлении/изменении автоматически пингуется IndexNow (быстрая индексация).

────────────────────────────────────────────────────────────────────
6. РЕКОМЕНДУЕМЫЙ СЦЕНАРИЙ
────────────────────────────────────────────────────────────────────
1) Один раз: подтвердить поля по реальному ответу API, заполнить alstyle-import.js.
2) Прогнать на 1 категории (напр. Видеонаблюдение) — проверить цены/фото на сайте.
3) Включить нужные категории в админке (галочка «Показывать»).
4) Поставить запуск по расписанию (cron / Render Cron Job): раз в день fullSync:true,
   в течение дня — лёгкие обновления остатков (fullSync:false).
5) Цены проверять: розница не ниже закупа, наценка корректная.

Готовый скрипт-заготовка — в файле  alstyle-import.js  (заполнить ключ, URL, поля).


────────────────────────────────────────────────────────────────────
ДОПОЛНИТЕЛЬНО: ПОЛНОЕ ДЕРЕВО КАТЕГОРИЙ
────────────────────────────────────────────────────────────────────
Скрипт  alstyle-categories.js  выгружает ПОЛНОЕ дерево всех категорий и
подкатегорий Al-Style из API в файл categories_full.txt (с чекбоксами для выбора).
Запуск: node alstyle-categories.js (предварительно вписать ключ и URL метода категорий).
Это даёт 100% актуальное дерево всех разделов — надёжнее ручного сбора с сайта.
