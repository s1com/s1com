ОБНОВЛЕНИЯ ЗА СЕГОДНЯ — всё в одном архиве
============================================

Залейте СОДЕРЖИМОЕ архива в GitHub (с сохранением папок), один раз закоммитьте,
потом Render → Manual Deploy, и в конце — Cron → Trigger Run.

ЧТО ВНУТРИ (14 файлов кода + этот README):
  db.js                              → корень
  server.js                          → корень
  lib/product-page.js                → папка lib/
  scripts/alstyle-import.js          → папка scripts/
  scripts/seed.js                    → папка scripts/
  scripts/backup.js                  → папка scripts/
  public/index.html                  → папка public/
  public/videonablyudenie.html       → папка public/
  public/setevoe.html                → папка public/

  admin/index.html                   → папка admin/

ЧТО ИЗМЕНИЛОСЬ ЗА ДЕНЬ:
  1) ДЕРЕВО КАТЕГОРИЙ (как у Al-Style). IP видеокамеры → Купольные/Цилиндрические/
     PTZ → 2/4/5/8 МП, Wi-Fi-ветка, видеорегистраторы по каналам. Категории
     хранятся по ID Al-Style (одинаковые «2 мегапиксельные» под разными типами
     больше не конфликтуют). Слева дерево с раскрытием по уровням, клик фильтрует.
  2) ХАРАКТЕРИСТИКИ. Описание парсится в 2-колоночный список характеристик
     (лимит описания поднят до 6000 символов).
  3) ПОЛНЫЕ ИМЕНА товаров. Берём полное название из Al-Style (а не короткий
     артикул); бренд не дублируется в заголовке.
  4) SEO. На всех страницах: микроразметка (BreadcrumbList, CollectionPage,
     FAQPage), видимый FAQ-аккордеон, блок «Смотрите также», SEO-тексты
     (Усть-Каменогорск/ВКО), lastmod в sitemap.
  5) Прочее за день: дашборд-аналитика, фильтры в выпадашке, галерея фото,
     остатки «в наличии», диагностика Al-Style API, WAL-безопасный бэкап.

ДЕПЛОЙ — по шагам:
  1) GitHub → Add file → Upload files → перетащите СОДЕРЖИМОЕ архива
     (db.js, server.js + папки lib/ scripts/ public/ admin/ tests/).
     Если заливаете по одному — кладите каждый файл в его папку из списка выше.
  2) Commit changes (в main).
  3) Render (servis-catalog) → Manual Deploy → Deploy latest commit → дождитесь Live.
  4) ⚠ ОБЯЗАТЕЛЬНО: Render → ваш Cron → Trigger Run (дерево и характеристики
     заполняются только при импорте).

⚠ ВАЖНО про категории (из-за перехода на дерево по ID):
  • До первого импорта раздел категорий будет пустым — это норма. Запустите Cron.
  • Старая таблица категорий пересоздаётся под новую схему — ручные настройки
     видимости сбросятся (заполнятся заново при импорте, всё видимо).
  • Ручное добавление категорий в админке отключено (дерево из импорта). Скрывать
     лишние ветки — галочкой «Показ» в админке → 🗂 Категории (сохраняется между
     импортами; скрытие родителя прячет и подкатегории).

ПРОВЕРКА что код залит:
  • Откройте /js/app.js, Ctrl+F → recomputePool (есть = новая версия на месте).
  • После импорта слева в «Видеонаблюдении» — дерево с раскрытием по уровням.

НЕ ВКЛЮЧЕНО (по желанию, к сайту не относится):
  • Старые мёртвые дубли в корне репозитория: app.js, index.html, style.css —
     можно удалить в GitHub (рабочие версии лежат в public/). На сайт не влияют.
