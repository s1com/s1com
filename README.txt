ВСЕ ПОСЛЕДНИЕ ИЗМЕНЕНИЯ — единый архив (для ВЕБ-сервиса)
=======================================================
Один деплой на веб-сервис. Импорт/крон запускать НЕ нужно.

СОДЕРЖИМОЕ (что вошло):
  db.js              — таблицы: suppliers, offers, category_map, price_rules,
                       match_queue, faq (+ поле amount у заявок). Сиды
                       поставщиков и FAQ. Всё аддитивно.
  server.js          — эндпоинты: /api/offers-sync, /api/admin/suppliers,
                       /api/faq + /api/admin/faq (CRUD), статусы заявок +
                       /api/admin/sales-stats (конверсия и сумма продаж).
  admin/index.html   — вкладки «🏭 Поставщики» и «❓ FAQ»; в заявках статусы
                       Новая/В работе/Заказ/Отказ + поле суммы; в Дашборде
                       блок «Сделки и конверсия» с переключателем периода.
  lib/product-page.js— характеристики больше не обрезаются (снят лимит 140).
  public/*.html (5)  — FAQ на страницах берётся из админки и обновляет
                       FAQPage-разметку (со статическим SEO-fallback).

ЧТО НЕ ВКЛЮЧЕНО (намеренно):
  • public/js/app.js и public/css/style.css — там мегаменю Аспро и верхние
    фильтры, которые ты не захотел. Текущий вид каталога остаётся как есть.
    Если нужно — скажи, добавлю отдельно.
  • scripts/alstyle-import.js — это крон, он уже задеплоен и работает
    (офферы синхронизируются). Трогать не нужно.

ДЕПЛОЙ (важно — заменить ВСЕ файлы):
 1) GitHub → Add file → Upload files → перетащить в корень:
       db.js, server.js
    и папки целиком:
       admin/   (index.html)
       lib/     (product-page.js)
       public/  (5 html)
    Сохраняя структуру папок. → Commit в main.
 2) Render → ВЕБ-сервис (servis-catalog) → Manual Deploy →
    «Clear build cache & deploy» (лучше с очисткой кэша) → дождаться Live.
 3) Открыть админку и сайт с Cmd+Shift+R.

ПРОВЕРЕНО: синтаксис всех файлов, JS админки, FAQ на 5 страницах,
все 11 таблиц БД создаются без конфликтов.
