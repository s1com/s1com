# 🚀 Развёртывание сайта — пошагово

Цель: поднять сайт на хостинге за ~30–60 минут. Нужен человек, знакомый с Git/хостингом.

## ВАРИАНТ 1 — Render.com (рекомендую, есть конфиг)

### Шаг 1. Подготовка
1. Создайте аккаунт на https://render.com
2. Залейте код в репозиторий на GitHub (или используйте «Deploy from Git»).

### Шаг 2. Создание сервиса
1. Render → New → Web Service → подключите репозиторий.
2. Render найдёт `render.yaml` и предложит настройки — подтвердите.
3. ВАЖНО: для постоянного диска (чтобы база и фото не пропадали) нужен план **Starter ($7/мес)**.

### Шаг 3. Переменные окружения (в дашборде Render → Environment)
Обязательные:
- `ADMIN_PASSWORD` — придумайте сложный пароль для админки
- `SITE_URL` — адрес сайта (сначала `https://servis-catalog.onrender.com`, потом ваш домен)

Автоматические (Render сгенерирует сам): `JWT_SECRET`, `IMPORT_TOKEN`.
Не забыть включено: `SEED_ON_EMPTY=true` (каталог загрузится сам при первом старте).

Для SEO (можно добавить после первого запуска):
- `INDEXNOW_KEY` — случайная строка 32 символа (можно сгенерировать: openssl rand -hex 16)
- `YANDEX_VERIFICATION`, `GOOGLE_VERIFICATION` — после добавления сайта в вебмастеры
- `YM_ID` — номер счётчика Яндекс.Метрики

### Шаг 4. Деплой
- Render соберёт и запустит автоматически. Каталог (270 товаров) загрузится сам.
- Проверьте: `https://ваш-адрес/health` → должно вернуть `{"status":"ok",...}`
- Откройте сайт, проверьте каталог. Админка: `/admin/`

### Шаг 5. Домен (опционально)
- Render → Settings → Custom Domain → добавьте servis-com.kz, пропишите DNS по инструкции Render.
- Обновите `SITE_URL` на реальный домен.

## ВАРИАНТ 2 — VPS (Timeweb, Beget, Selectel)
1. Сервер с Ubuntu, установите Node 20: `apt install nodejs npm`
2. Скопируйте код, `cd servis-app`, `npm ci || npm install`
3. Создайте `.env` (см. .env.example), задайте секреты.
4. `npm run seed` (загрузить каталог) — один раз.
5. Запуск через pm2: `npm i -g pm2 && pm2 start server.js --name servis && pm2 save && pm2 startup`
6. Nginx как реверс-прокси + Let's Encrypt для HTTPS (стандартная настройка).

## ВАРИАНТ 3 — Docker (любой хостинг с Docker)
```
docker build -t servis-catalog .
docker run -d -p 3000:3000 --env-file .env -v /host/data:/data --name servis servis-catalog
```

## ПОСЛЕ ЗАПУСКА (SEO, по чек-листу)
1. Яндекс.Вебмастер (webmaster.yandex.ru): добавить сайт → мета-тег → `YANDEX_VERIFICATION` → подтвердить → добавить sitemap `https://домен/sitemap.xml`
2. Google Search Console: аналогично → `GOOGLE_VERIFICATION` → sitemap.
3. Задать `INDEXNOW_KEY` — новые товары начнут индексироваться мгновенно.
4. Зарегистрировать компанию в **Яндекс.Бизнес** и **Google Business Profile** (для локальной выдачи и карт).

## ИНТЕГРАЦИЯ С 1С (когда будете готовы)
1С шлёт прайс с остатками на `POST https://домен/api/import` с заголовком
`Authorization: Bearer <IMPORT_TOKEN>`. Формат и пример — в README.md.
До настройки автообмена можно вести каталог через **админку** (ручное редактирование + Импорт из Excel/CSV).
